# v1.16.0 医療費控除の本格対応

## 目的

確定申告書「医療費控除の明細書」（国税庁様式）に直接転記できるレベルへ、医療費控除の管理機能を精緻化する。現状の `tax_tag=MEDICAL` 単純集計から脱却し、**受診者・医療機関・区分・補填額**を構造化保存し、控除額自動計算と申告書様式準拠 CSV を出力する。

家計簿としての既存 Transaction 入力フロー（v1.13.0 互換）は壊さない。医療費機能を独立テーブルに切り出し、必要に応じて Transaction と紐付ける「拡張」方式を採用する。

## 設計（合意済み）

### ① データモデル: `MedicalExpense.transaction` (nullable FK on MedicalExpense 側)

- 新規モデル `MedicalExpense` に Transaction への nullable FK を持たせる
- Transaction テーブルには列を追加しない（既存スキーマ完全保護、v1.2.0 Transfer 分離思想を踏襲）
- 1 取引 1 医療費を `UniqueConstraint('transaction')` で担保
- 家計簿に載らない医療費（保険組合の事後請求、世帯メンバーの別決済 等）は `transaction=NULL` で登録可

### ② 入力 UI: 取引フォーム条件展開 + 専用一覧/編集ページの両立

| 動線 | 用途 |
|---|---|
| 取引フォーム内に HTMX 条件展開（カテゴリの `tax_tag=MEDICAL` 選択時） | 都度入力（入力漏れ最小化） |
| `/medical-expenses/?year=YYYY` 専用ページ | 後追い編集 + Transaction 無しの医療費登録 + 申告期一括レビュー |

### ③ 総所得金額: `AnnualIncomeSnapshot` モデルで年単位保存

- 控除額計算式: `控除額 = 合計支払額 − 補填額合計 − min(100,000, 総所得金額 × 5%)`
- 総所得 200 万円超 → 一律 10 万円ライン
- 総所得 200 万円以下 → 5% ライン
- 年単位で 1 件保存（year / gross_income / notes）。再表示時の再入力不要、admin 編集可
- 未入力時は「一律 10 万円ライン」で暫定計算 + UI に警告表示

### ④ 区分: 国税庁様式準拠 4 区分

| Enum | UI ラベル | 国税庁明細書様式 |
|---|---|---|
| `TREATMENT` | 診療・治療（病院・歯科） | 診療・治療 |
| `MEDICINE` | 医薬品（処方薬・市販薬） | 医薬品購入 |
| `CARE_INSURANCE` | 介護保険サービス | 介護保険サービス |
| `OTHER` | その他（通院交通費 等） | その他の医療費 |

CSV 出力は申告書転記がマッピング無しで済む列順 / ラベル順。

## スコープ

### モデル定義

#### `MedicalExpense`

```python
class MedicalExpense(TimeStampedModel):
    class Category(models.TextChoices):
        TREATMENT = 'treatment', '診療・治療（病院・歯科）'
        MEDICINE = 'medicine', '医薬品（処方薬・市販薬）'
        CARE_INSURANCE = 'care_insurance', '介護保険サービス'
        OTHER = 'other', 'その他（通院交通費 等）'

    transaction = models.ForeignKey(
        'Transaction',
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name='medical_expense_set',
        verbose_name='紐付け取引',
    )
    paid_date = models.DateField('支払日')
    patient = models.CharField('受診者', max_length=50)
    provider = models.CharField('医療機関・薬局名', max_length=120)
    category = models.CharField(
        '区分',
        max_length=20,
        choices=Category.choices,
        db_index=True,
    )
    amount = models.IntegerField(
        '支払額',
        validators=[MinValueValidator(1)],
    )
    reimbursement = models.IntegerField(
        '補填額',
        default=0,
        validators=[MinValueValidator(0)],
        help_text='保険金・出産育児一時金・高額療養費等で補填された金額',
    )
    notes = models.TextField('メモ', blank=True)

    class Meta:
        verbose_name = '医療費'
        verbose_name_plural = '医療費'
        ordering = ['-paid_date', '-id']
        constraints = [
            models.UniqueConstraint(
                fields=['transaction'],
                condition=models.Q(transaction__isnull=False),
                name='uniq_medical_expense_per_transaction',
            ),
        ]

    def clean(self):
        super().clean()
        if self.reimbursement > self.amount:
            raise ValidationError({
                'reimbursement': '補填額は支払額以下で入力してください。',
            })
        if self.transaction_id:
            tx = self.transaction
            if tx.category.tax_tag != Category.TaxTag.MEDICAL:
                raise ValidationError({
                    'transaction': '紐付け先取引のカテゴリは「医療費控除」タグである必要があります。',
                })
            if tx.amount != self.amount:
                raise ValidationError({
                    'amount': '紐付け取引の金額と支払額が一致していません。',
                })
            if tx.date != self.paid_date:
                raise ValidationError({
                    'paid_date': '紐付け取引の日付と支払日が一致していません。',
                })
```

### CSV 仕様

- ファイル名: `medical-expenses-{year}.csv`
- 文字コード: UTF-8 BOM 付き（既存 export と同じ）
- 列順（**国税庁「医療費控除の明細書」様式準拠**）:
  1. 医療を受けた方の氏名 (`patient`)
  2. 病院・薬局などの支払先の名称 (`provider`)
  3. 医療費の区分 (`category` の日本語表示)
  4. 支払った医療費の額 (`amount`)
  5. 左のうち、補填される金額 (`reimbursement`)
  6. 差引額 (`amount - reimbursement`)
  7. 支払日 (`paid_date`, 様式外だが照合用に追加)
- フッター行: 合計 / 控除額計算
- CSV injection 対策: 既存 `csv_safe_row` 経由

## 控除額計算ロジック詳細

```python
def calculate_medical_deduction(year: int) -> dict:
    """医療費控除額を計算する。

    Returns:
        {
            'total_paid': int,         # 支払合計
            'total_reimbursement': int, # 補填合計
            'net_paid': int,           # 差引合計 = total_paid - total_reimbursement
            'threshold': int,          # 控除基準額 = min(100000, gross_income * 5%)
            'deduction': int,          # 控除額 = max(0, net_paid - threshold)
            'gross_income_known': bool, # 総所得が登録されているか
        }
    """
    expenses = MedicalExpense.objects.filter(paid_date__year=year)
    total_paid = expenses.aggregate(s=Sum('amount'))['s'] or 0
    total_reimbursement = expenses.aggregate(s=Sum('reimbursement'))['s'] or 0
    net_paid = total_paid - total_reimbursement

    snapshot = AnnualIncomeSnapshot.objects.filter(year=year).first()
    if snapshot:
        threshold = min(100_000, snapshot.gross_income * 5 // 100)
        gross_income_known = True
    else:
        threshold = 100_000  # 暫定
        gross_income_known = False

    deduction = max(0, net_paid - threshold)
    return {
        'total_paid': total_paid,
        'total_reimbursement': total_reimbursement,
        'net_paid': net_paid,
        'threshold': threshold,
        'deduction': deduction,
        'gross_income_known': gross_income_known,
    }
```

整数除算で 5% を計算する点に注意（円単位、端数切り捨て）。

## セキュリティ / データ保護

- 全 view `@login_required`（Cloudflare Access 前段 + Django ログインの二重認証は維持）
- 入力値は `MedicalExpenseForm` / `AnnualIncomeSnapshotForm` で Django form validation を通す
- `patient` / `provider` は CharField のため XSS 自動エスケープ（template autoescape 前提、`safe` フィルター不使用）
- CSV injection 対策: `csv_safe_row` 経由（`=`, `+`, `-`, `@` 始まりセルにシングルクオート前置）
- `MedicalExpense.delete` は POST のみ受付（CSRF 保護）
- AuditLog: 新規 source `medical_expense_create` / `medical_expense_update` / `medical_expense_delete` / `income_snapshot_save` を追加し、操作系を記録
- `patient` フィールドは個人情報なので、CSV ダウンロード以外で外部送信されないことを確認

## 想定運用フロー

```
1. 年初: /settings/income-snapshots/ で前年の総所得を登録
   （源泉徴収票の「給与所得控除後の金額」欄）
2. 通年: 病院・薬局支払い時に取引入力 → MEDICAL カテゴリ選択時に展開される
         「受診者 / 医療機関 / 区分 / 補填額」をその場で入力
3. 保険組合からの還付通知: /medical-expenses/<id>/edit/ で補填額を後追い更新
4. 家計簿に載らない医療費（世帯メンバー別決済 等）:
   /medical-expenses/new/ で transaction 無しで登録
5. 翌年 2-3 月: /medical-expenses/?year=前年 で集計確認 →
   CSV ダウンロード → 申告書「医療費控除の明細書」へ転記
6. 控除額カードの数値を申告書第二表 ⑩医療費控除 にそのまま記入
```
