# v1.9.0 PWA 化

## 目的

iPad / スマートフォンで「ホーム画面に追加」してアプリ風起動でき、ネットワーク切断時にも最低限の表示ができる状態にする。LAN 公開（v1.5.0）と Cloudflare Tunnel 経由公開のどちらでも動作するように、特定オリジンに依存しない形で実装する。

## 仕様

### 範囲

- `manifest.webmanifest` を root scope で配信
- Service Worker（`/sw.js`）を root scope で登録
- iOS（Safari）向け meta タグと apple-touch-icon を提供
- オフライン時の簡易フォールバック画面
- 既存ページ・既存 API への破壊的変更なし

### manifest

`/manifest.webmanifest` を Django view 経由で配信する（`Content-Type: application/manifest+json`）。

```json
{
  "name": "BudgetBook",
  "short_name": "BudgetBook",
  "description": "個人家計簿アプリ",
  "start_url": "/",
  "scope": "/",
  "display": "standalone",
  "orientation": "any",
  "background_color": "#ffffff",
  "theme_color": "#2563eb",
  "icons": [
    {"src": "/static/icons/icon.svg",  "sizes": "any", "type": "image/svg+xml", "purpose": "any"},
    {"src": "/static/icons/icon-192.png", "sizes": "192x192", "type": "image/png", "purpose": "any"},
    {"src": "/static/icons/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "any"},
    {"src": "/static/icons/icon-mask-512.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable"}
  ]
}
```

- `theme_color` は既存ブランド色（`#2563eb` = base.html の favicon 背景色）と一致させる。
- `display: standalone` でアドレスバーを隠す。

### アイコン

- ベース: SVG 1 枚（`#2563eb` 背景 + 白の "B" 文字）
- PNG: 192 / 512 / maskable 512 / apple-touch 180 — Pillow で SVG からビルド時生成するのは依存追加になるため、**初期は静的 PNG をリポジトリ同梱**する（軽量、数十 KB）。
- icon ファイルは `budgetbook/static/icons/` 配下に配置。

### Service Worker

- ファイルパス: `/sw.js`（root scope を取れるよう view 経由で配信、`Service-Worker-Allowed: /` ヘッダは不要）
- 戦略:
  - **App shell**: `/`, `/static/css/style.css`, `/static/js/htmx_config.js` などをインストール時にキャッシュ（precache）
  - **静的アセット (`/static/`)**: cache-first
  - **HTML ページ**: network-first、失敗時はキャッシュの app shell → さらに失敗時は `/offline` フォールバック
  - **POST / PUT / DELETE / API**: 一切インターセプトしない（家計データを古いキャッシュで上書きしないため）
- バージョン番号 (`CACHE_VERSION`) を SW 内に埋め込み、`STATIC_VERSION` と連動させる。`activate` 時に旧キャッシュを削除。

### 登録スクリプト

`base.html` に CSP nonce 付きで以下を追加（`navigator.serviceWorker` が存在しブラウザ対応がある場合のみ登録）:

```html
<script nonce="{{ csp_nonce }}">
  if ('serviceWorker' in navigator) {
    window.addEventListener('load', function () {
      navigator.serviceWorker.register('/sw.js', { scope: '/' });
    });
  }
</script>
```

別ファイル `static/js/pwa_register.js` に切り出して `<script nonce="..." src="..." defer>` で読み込む方針とする（既存パターンと整合）。

### オフラインフォールバック

`/offline` view を追加。シンプルな案内 + 「再読み込み」ボタンのみ。SW がネットワーク失敗時に返す。

### iOS 対応 meta タグ

`base.html` の `<head>` に追加:

```html
<link rel="manifest" href="/manifest.webmanifest">
<meta name="theme-color" content="#2563eb">
<link rel="apple-touch-icon" href="/static/icons/apple-touch-icon.png">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="apple-mobile-web-app-title" content="BudgetBook">
```

### CSP / セキュリティ

- 既存 CSP (`script-src 'self' 'nonce-...'`) でカバー済。Service Worker 自身は `'self'` でロードされるので追加設定不要。
- 既存の `connect-src 'self'` のままで OK（外部送信はしない）。
- manifest / sw は **未ログインでもアクセス可能**にする（PWA インストール時にブラウザがログイン前に取得するため）。ただし機微データは含めない。
- ServiceWorker のキャッシュ対象から `/accounting/`, `/balance-sheet/`, `/transactions/` などログイン必須ページの **HTML は除外** する（precache はせず、network-first のみ）。
- `/healthz` のキャッシュは禁止（運用観測用）。

### HTTPS 要件（注意事項）

- ServiceWorker は **HTTPS または `localhost`** でのみ登録できる。
- LAN 内 HTTP アクセス（`http://192.168.1.10:8010/`）では **iPad / スマホで PWA インストール不可**。
- 解決策（オペレーション側）:
  - Cloudflare Tunnel 経由でアクセスすれば HTTPS になり PWA 利用可能。
  - LAN 内利用は「ブラウザでブックマーク」相当に留まる（ホーム画面追加だけは Safari の機能で可能だが、SW は動かない）。
- この制約は `OPERATIONS_GUIDE.md` に追記する。

## モデル変更

- なし。migration なし。

## 非対象

- Push 通知 / Background Sync
- IndexedDB を使った取引データの完全オフライン化
- 多言語 manifest
- Workbox など外部ライブラリ導入（依存追加を避ける）
- PWA install promotion バナーの自前実装（ブラウザ標準に任せる）

## 受け入れ条件

- `/manifest.webmanifest` が 200 / `application/manifest+json` で返る
- `/sw.js` が 200 / `application/javascript` で返る
- `/offline` が 200
- `base.html` に `<link rel="manifest">` と `<link rel="apple-touch-icon">` が含まれる
- 既存 317 テストが緑のまま
- HTTPS 環境 (localhost / Cloudflare Tunnel) で Chrome の `Application > Manifest` / `Service Workers` が緑表示
- Lighthouse の "PWA Installable" 条件を満たす

## テスト観点

`budgetbook/ledger/tests/test_pwa.py` を新規追加。

- manifest 200、Content-Type、`start_url`/`scope`/`name`/`icons` の存在
- sw.js 200、Content-Type、`Service-Worker-Allowed` ヘッダがあれば検査
- /offline 200、CSP ヘッダがある
- base.html に必要な meta / link が含まれる（任意ページの GET レスポンスで検査）
- manifest と sw が **未ログインでも 200**（PWA インストール時の挙動）
- 既存の `/accounting/` 等のログイン必須はそのまま 302
- CSP の `script-src` に `pwa_register.js` が nonce 経由で読み込まれること（HTML に nonce 付き `<script src>` が含まれること）

## セキュリティ影響

- 公開エンドポイント追加: `/manifest.webmanifest`, `/sw.js`, `/offline`。いずれも機微データを含まない静的相当の応答のみ。
- Service Worker は自オリジン上でキャッシュ・配信を行う。同オリジン以外への送信は行わない。
- ログイン必須ページの HTML は precache 対象外とし、network-first のみで取得（古いキャッシュで個人データを上書き表示しない）。

## DB 影響

- なし。

## 既存データへの影響

- なし。

## ロールバック方針

- 機能を外す場合: `base.html` の link / meta / register script を削除、URL 3 つを外せば終了。
- 既にインストールされた SW が残るユーザーには `self.registration.unregister()` を返す **キルスイッチ用 sw.js** を用意可能（運用必要時のみ）。
