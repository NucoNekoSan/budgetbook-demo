# v1.3.0 Income Expense Ratio

Status: 実装完了

## 目的

収入に対して、カテゴリ別にどのくらい支出したかを比率で確認できるようにする。
また、実運用上は店舗・購入先ベースのカテゴリが細かく分かれているため、支出構成グラフでは複数カテゴリを任意の集計グループとしてまとめて確認できるようにする。

## 仕様

- 支出構成ページに「収入に対する支出比率」セクションを追加する。
- 月間収入を母数にする。
- 支出カテゴリごとの金額と割合を表示する。
- 月間収入 > 月間支出 の場合は、差額を「残額」として円グラフに含める。
- 月間支出 > 月間収入 の場合は、支出超過であることが分かる表示を追加する。
- 月間収入が 0 の場合はグラフを表示せず、計算不能の空状態を表示する。
- `Transfer` は集計に含めない。

## 支出カテゴリグループ

### 背景

現在、以下のようなカテゴリがある。

- スーパー
- 百貨店
- コンビニ
- イオン・マックスバリュー
- Co-op
- ツルハドラッグ
- ドラッグストア

これらは店舗・購入先としては別カテゴリだが、実際には食料品、飲料、普段使う日用品の購入が多い。そのため、円グラフで見ると支出が細かく分散し、生活費の実態を把握しづらい。

### 方針

既存の `Category` は維持し、取引入力時のカテゴリも細かいまま使えるようにする。

円グラフや支出比率表では、複数の `Category` を任意の「支出カテゴリグループ」にまとめて表示できるようにする。

例:

```text
グループ名: 食品・日用品
対象カテゴリ:
- スーパー
- コンビニ
- イオン・マックスバリュー
- Co-op
- ツルハドラッグ
- ドラッグストア
```

百貨店は購入内容が食品・日用品とは限らないため、初期設定では別グループまたは未分類扱いにしてよい。

### 実装候補

推奨は、表示集計用のモデルを追加する方式。

- `ExpenseGroup`
  - `name`
  - `is_active`
  - `sort_order`
- `ExpenseGroupCategory`
  - `group`
  - `category`

または `ExpenseGroup.categories = ManyToManyField(Category)` でもよい。既存コードとの相性を見て判断する。

制約:

- グループ対象は支出カテゴリのみ。
- 収入カテゴリはグループに含めない。
- 1つのカテゴリは原則1つのグループにのみ所属させる。
- グループ未設定のカテゴリは、従来通りカテゴリ名で個別表示する。

### 集計仕様

- 支出構成ページの円グラフは、グループ設定があるカテゴリをグループ名で合算する。
- グループ未設定カテゴリはカテゴリ単位で表示する。
- 表には、必要に応じてグループ行と内訳カテゴリを表示できるようにする。
- 既存の取引データは変更しない。
- `Transfer` は含めない。

### 管理仕様

設定画面または Django admin で、カテゴリグループと所属カテゴリを管理できるようにする。

v1.3.0 で画面実装の負荷が大きい場合、まずは Django admin 管理のみでよい。その場合も、将来の設定画面追加を妨げないモデル設計にする。

## 受け入れ条件

- 収入 100,000 円、食費 30,000 円、交通費 10,000 円の場合、食費 30%、交通費 10%、残額 60% として確認できる。
- 支出超過月でも誤解のない表示になる。
- スーパー 10,000 円、コンビニ 5,000 円、Co-op 8,000 円、ドラッグストア 3,000 円が「食品・日用品」グループに所属している場合、円グラフでは「食品・日用品 26,000 円」として合算表示される。
- グループ未設定カテゴリは、従来通り個別カテゴリとして表示される。
- 百貨店は初期設定では食品・日用品に自動合算しない。
- 既存の取引カテゴリは変更されない。
- `python manage.py test ledger` が通る。
