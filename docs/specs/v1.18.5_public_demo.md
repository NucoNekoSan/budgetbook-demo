# v1.18.5 Public Demo / Self-host Distribution

> **Status**: ✅ **実装完了（2026-05-14）**

## 目的

家計簿アプリを **GitHub 上のポートフォリオ + セルフホスト配布** として一般家庭向けに公開する。private 運用とは独立した repo として、実データを含まない demo モードと配布用 script 群を整備する。

## スコープ

### 1. DEMO_MODE 環境変数

| 環境変数 | デフォルト | 効果 |
|---|---|---|
| `DEMO_MODE` | `0` | `1` で「デモデータです」バナー + mutation ブロック |
| `DEMO_ALLOW_WRITES` | `0` | `1` で `DEMO_MODE` 中でも書き込みを許可（preview / sandbox 用） |
| `DEMO_AUTO_LOGIN` | `0` | （将来用、未実装）`1` で demo ユーザー自動ログイン |

### 2. `DemoModeWriteBlockMiddleware`

`config.middleware.DemoModeWriteBlockMiddleware` を MIDDLEWARE の末尾近くに追加。

動作:
- `DEMO_MODE=0` のとき: 何もしない（通常運用）
- `DEMO_MODE=1 & DEMO_ALLOW_WRITES=1` のとき: 書き込み許可（preview sandbox）
- `DEMO_MODE=1 & DEMO_ALLOW_WRITES=0` のとき:
  - GET / HEAD / OPTIONS / TRACE: 通過
  - その他（POST / PUT / PATCH / DELETE）: `/accounts/login/` 等を除き `403 demo_blocked` を返す

### 3. デモバナー

`base.html` の `<body>` 直下にバナーを表示（`DEMO_MODE=True` のときのみ）。スタイルは `static/css/style.css` の `.demo-banner` 定義。

### 4. `seed_demo_data` 管理コマンド

`python manage.py seed_demo_data --reset` で以下を生成:

- 口座 8 件（現金 / 普通預金 / 貯蓄預金 / 証券口座 / 電子マネー / 住宅ローン / 自動車ローン / クレジットカード）
- カテゴリ 21 件（収入 3 / 支出 18、税控除タグ付きあり）
- LoanProfile 2 件（住宅ローン / 自動車ローン、annual_rate_bp 設定）
- 取引 55 件（過去 3 ヶ月分の典型的家計）
- 振替 2 件（クレジット引落 + 積立貯蓄）
- SectionBudget 10 件（当月分）
- MedicalExpense 6 件（前年分、4 人家族想定）
- InsurancePremium 4 件（前年分、生保 3 枠 + 地震）
- AnnualIncomeSnapshot 1 件（前年総所得 ¥4,500,000）
- demo ユーザー（`demo` / `demo`）+ admin ユーザー（`admin` / `admin`）

`--seed N` でランダム seed 指定（再現性確保、デフォルト 42）。

### 5. Windows 用ワンクリック script

- `start-windows.bat`: Docker 起動 → migrate → 初回 seed → ブラウザ起動
- `stop-windows.bat`: コンテナ停止
- `reset-demo-data.bat`: 既存 DB をバックアップ後、demo データ再投入
- `backup-data.bat`: `scripts/backup_budgetbook.ps1` または手動 copy でバックアップ

### 6. Public 向けドキュメント

- `README.md`: 機能紹介 / セルフホスト手順 / Tech stack / License
- `LICENSE`: MIT License

### 7. 個人情報除去（一括 sed bulk replace）

private 運用時に使われていた具体名（実銀行名 / 実カード名 / 個人ドメイン / LAN IP / 個人 email 等）を一般化された placeholder（`普通預金A` / `クレジットカードA` / `example.com` / `192.168.1.10` / `demo@example.com` 等）へ一括 `sed` 置換した。

例外: `test_demo_mode.py::test_seed_does_not_use_real_personal_categories` は **元の禁止文字列リスト**を内部に保持（regression check として機能、置換対象外）。詳細な置換対応は `git log` ではなく作業時の grep スクリプトで確認する。

## DB 影響

- migration なし（DEMO_MODE は env-var フラグのみ）
- `seed_demo_data` は新規 user / 口座 / カテゴリ / 取引等を投入するが、`--reset` 必須で破壊的動作を明示

## セキュリティ / データ保護

- DEMO_MODE は **defense-in-depth**：仮に demo サイトが認証突破されても mutation を 403 で防ぐ
- `DEMO_ALLOW_WRITES=1` を使う場合は短周期 (例: 30 分ごと) の seed 自動 reset と組み合わせる前提
- public repo の git history は **新規 `git init` から始める**（private repo の履歴を引き継がない、過去コミットに個人情報残存リスクを完全排除）
