# v1.12.0 LoanProfile 元金返済 Transfer 自動化

## 目的

`LoanProfile` に `source_account`（引落元の資産口座）と既存 `monthly_payment` / `payment_day` を組み合わせ、毎月の返済 Transfer を自動生成する。v1.11.0 の利息自動計上と合わせて、リボ運用の月次手入力をゼロにする。

## 設計判断（合意済み）

- **monthly_payment の解釈**: 銀行から引かれる**総額**。Transfer 1 件で銀行→負債口座へそのまま移動
- **計上日**: `LoanProfile.payment_day`（1〜31）。0 または無効値は月末にフォールバック。月の日数を超える日付（例: 2月30日）は月末
- **source_account**: LoanProfile に **nullable FK** を追加。設定済みのみが対象
- **0% ローン**: `monthly_payment > 0` かつ source_account が設定されていれば、`annual_rate_bp=0` でも処理（元金返済はある）

## スコープ

### 対象

- migration: `LoanProfile.source_account` (FK to Account, null=True, blank=True, on_delete=SET_NULL)
- `LoanProfileForm` に source_account フィールド追加（資産口座のみ選択肢）
- `python manage.py accrue_loan_principal --month YYYY-MM [--apply] [--account 口座名]` 新規追加
- 対象抽出: `monthly_payment > 0 AND source_account__isnull=False AND account.is_active AND source_account.is_active`
- Transfer 1 件 / LoanProfile、from=source_account, to=loan.account, amount=monthly_payment
- 摘要: `f'{account.name} {YYYY-MM} 返済'`
- AuditLog 記録（`source=accrue_loan_principal`, metadata に month/account/source_account/amount）
- 同月同 (from, to, source=accrue_loan_principal) の Transfer が既にあれば **エラー終了**（二重計上防止）
- MonthlyClosing 済み月は拒否

### 非対象

- 元金 / 利息の内訳分解（複利計算の高度化）
- monthly_payment の自動更新（残債減少に応じた目安額の見直し）
- 完済時の自動 deactivate
- UI 上での実行（CLI のみ。利息コマンドと統一）

## 計算ロジック

```
amount = LoanProfile.monthly_payment  # そのまま使う、計算なし
date   = date(year, month, min(payment_day, last_day_of_month))
         if payment_day in 1..31 else last_day_of_month
```

利息側のロジックには触れない（独立してインポート可能）。

## CLI 仕様

```bash
# 当月を dry-run
docker compose exec budgetbook python manage.py accrue_loan_principal

# 月指定で dry-run
docker compose exec budgetbook python manage.py accrue_loan_principal --month 2026-06

# 確定実行
docker compose exec budgetbook python manage.py accrue_loan_principal --month 2026-06 --apply

# 特定口座のみ
docker compose exec budgetbook python manage.py accrue_loan_principal --month 2026-06 --account クレジットカードA --apply
```

出力例:

```

  クレジットカードA  普通預金A → クレジットカードA  2026-06-27  ¥30,000

  分割返済B        普通預金A → 分割返済B        2026-06-25  ¥5,000
合計返済: ¥45,000
※ --apply で確定します。今は何も書き込んでいません。
```

## DB 影響 / migration

| 項目 | 内容 |
|---|---|
| 追加カラム | `LoanProfile.source_account_id` (BigInteger FK, nullable) |
| 既存データ | すべての行で `source_account=NULL` でスタート。表示・既存ロジック影響なし |
| 削除カラム | なし |
| データ変換 | なし |
| ロールバック | `migrate ledger 0012` で安全に戻せる。新規 nullable カラムなので既存行は不変 |
| バックアップ | apply 前に `data/db.sqlite3` を `backup/` にコピー（運用ルール通り） |

## セキュリティ / データ保護

- CLI のみ。Web 公開面なし
- Transfer 自動生成は MonthlyClosing 済み月で拒否
- 二重計上を AuditLog metadata で識別して拒否
- `transaction.atomic` で部分失敗時の中途半端な状態を防止
