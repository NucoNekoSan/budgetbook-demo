# v1.17.0 生命保険料控除・地震保険料控除

> **Status**: ✅ **確定済**
> 起草: 2026-05-14
> レビュー完了: 2026-05-14（UA-1〜UA-6 全て現案採用）


## 目的

確定申告書 第二表「所得から差し引かれる金額に関する事項」の **生命保険料控除欄**（一般生命 / 介護医療 / 個人年金 の 3 枠を合算）および **地震保険料控除欄** に直接転記できる数値を、保険会社から届く「控除証明書」の入力から自動計算する。

v1.16.0 医療費控除と同様、Transaction とは独立した新規モデル `InsurancePremium` で管理する。保険料の銀行引落取引（`tax_tag` 連携の有無に関わらず）は壊さない。

給与所得者で **年末調整で控除証明書を会社に提出済** のケースは確定申告に再記載不要のため、年調済フラグで除外できるようにする。

## 設計（合意済み）

### ① データモデル: 独立テーブル `InsurancePremium`

- 新規モデル `InsurancePremium` を独立テーブルとして追加（Transaction とは紐付けない）
  - 保険料の引落取引は通常 1 件 / 月 × 12 件で年間総額と一致しない（年払い・前納など）
  - 控除証明書の数値（年間払込予定額 or 申告額）が正規の控除計算入力
  - Transaction との 1 対 1 紐付けは v1.16.0 のような利益がない
- Transaction テーブルには列を追加しない（既存スキーマ完全保護）

### ② 入力 UI: 専用ページのみ

- v1.16.0 のような取引フォーム条件展開は **不採用**
- 理由: 入力タイミングが年 1 回（控除証明書到着時 = 10〜11 月頃）かつ取引と数値が一致しないため、取引フォームでの誘導は誤入力を増やす
- `/insurance-premiums/?year=YYYY` 専用ページで CRUD

### ③ 計算ロジック: 国税庁公式式を 1:1 実装

- 各枠の控除額 → 生命保険料控除合算（上限 12 万円） + 地震保険料控除（上限 5 万円）を独立計算
- 新旧混在時は **3 方式の最大値を自動選択**（国税庁ルール準拠、ユーザー選択不要）
- 純粋関数として実装し、引数で全入力を受け取る（テスト容易）

### ④ 区分: 国税庁様式準拠 4 枠（旧長期損害保険は v1.17 では非対象）

| Enum | UI ラベル | 新旧契約区分 |
|---|---|---|
| `LIFE_GENERAL` | 一般生命保険料 | 新契約 / 旧契約 両方ありうる |
| `LIFE_CARE_MEDICAL` | 介護医療保険料 | **新契約のみ**（2012/1/1 新設） |
| `LIFE_ANNUITY` | 個人年金保険料 | 新契約 / 旧契約 両方ありうる |
| `EARTHQUAKE` | 地震保険料 | 新旧区分なし（一律ルール） |

旧長期損害保険料は v1.17.0 では対象外（非対象セクション参照）。

```python
class InsurancePremium(TimeStampedModel):
    class Category(models.TextChoices):
        LIFE_GENERAL = 'life_general', '一般生命保険料'
        LIFE_CARE_MEDICAL = 'life_care_medical', '介護医療保険料'
        LIFE_ANNUITY = 'life_annuity', '個人年金保険料'
        EARTHQUAKE = 'earthquake', '地震保険料'

    class ContractType(models.TextChoices):
        NEW = 'new', '新契約（2012/1/1 以降）'
        OLD = 'old', '旧契約（2011/12/31 以前）'

    year = models.PositiveIntegerField('対象年', db_index=True)
    category = models.CharField(
        '区分',
        max_length=20,
        choices=Category.choices,
        db_index=True,
    )
    contract_type = models.CharField(
        '契約区分',
        max_length=10,
        choices=ContractType.choices,
        default=ContractType.NEW,
    )
    insurer = models.CharField('保険会社名', max_length=120)
    policy_number = models.CharField(
        '証券番号',
        max_length=60,
        blank=True,
        help_text='控除証明書の証券番号（任意、本人識別用）',
    )
    annual_amount = models.IntegerField(
        '年間支払保険料',
        validators=[MinValueValidator(1)],
        help_text='控除証明書の「申告額」または「年間払込予定額」',
    )
    submitted_in_year_end_adjustment = models.BooleanField(
        '年末調整で提出済',
        default=False,
        help_text='ON にすると確定申告レポートでは除外される',
    )
    notes = models.TextField('メモ', blank=True)

    class Meta:
        verbose_name = '保険料控除'
        verbose_name_plural = '保険料控除'
        ordering = ['-year', 'category', '-id']
        indexes = [
            models.Index(fields=['year', 'category']),
        ]

    def clean(self):
        super().clean()
        # 介護医療保険料は新契約のみ存在（2012/1/1 新設）
        if (
            self.category == self.Category.LIFE_CARE_MEDICAL
            and self.contract_type == self.ContractType.OLD
        ):
            raise ValidationError({
                'contract_type': '介護医療保険料は新契約のみが対象です（2012/1/1 新設）。',
            })
        # 地震保険料は新旧区分の意味なし → NEW に正規化
        if self.category == self.Category.EARTHQUAKE:
            self.contract_type = self.ContractType.NEW
```

**設計判断 (M-1)**: `contract_type` を地震保険料でも NEW に正規化することで、計算時の分岐を簡素化。UI 側では地震カテゴリ選択時に契約区分セレクトを disabled にする。

## 控除額計算ロジック詳細

国税庁公式の計算式を **整数除算（端数切り捨て）** で 1:1 実装する。

### 各枠 1 件分の控除額（新契約）

```python
def calc_life_new(amount: int) -> int:
    """新契約 (2012/1/1 以降) の 1 枠分の控除額。上限 4 万円。"""
    if amount <= 20_000:
        return amount
    elif amount <= 40_000:
        return amount // 2 + 10_000
    elif amount <= 80_000:
        return amount // 4 + 20_000
    else:
        return 40_000
```

### 各枠 1 件分の控除額（旧契約）

```python
def calc_life_old(amount: int) -> int:
    """旧契約 (2011/12/31 以前) の 1 枠分の控除額。上限 5 万円。"""
    if amount <= 25_000:
        return amount
    elif amount <= 50_000:
        return amount // 2 + 12_500
    elif amount <= 100_000:
        return amount // 4 + 25_000
    else:
        return 50_000
```

**注意**: `amount // 2 + 12_500` の `12_500` は整数（25_000 / 2）。Python `//` は床関数なので、25,001 円なら `25_001 // 2 + 12500 = 12_500 + 12_500 = 25_000` で正しい。

### 1 枠の控除額（新旧混在時は有利選択）

```python
def calc_life_category(new_total: int, old_total: int) -> int:
    """1 枠（一般 or 介護医療 or 個人年金）の控除額。
    新旧両方ある場合は 3 方式の最大値を選ぶ（国税庁ルール）。

    Args:
        new_total: その枠で新契約の年間支払額合計
        old_total: その枠で旧契約の年間支払額合計

    Returns:
        その枠の控除額（その枠の上限は 40,000 or 50,000）
    """
    if new_total == 0 and old_total == 0:
        return 0
    if new_total == 0:
        return calc_life_old(old_total)
    if old_total == 0:
        return calc_life_new(new_total)

    # 新旧混在: 国税庁ルールで以下 3 方式から最大値
    method_new_only = calc_life_new(new_total)
    method_old_only = calc_life_old(old_total)
    method_combined_new_rule = calc_life_new(new_total + old_total)  # 上限 4 万円
    return max(method_new_only, method_old_only, method_combined_new_rule)
```

### 生命保険料控除合計（上限 12 万円）

```python
def calc_life_total(year: int, exclude_year_end_adjusted: bool = False) -> dict:
    """その年の生命保険料控除合計。上限 12 万円。"""
    qs = InsurancePremium.objects.filter(year=year).exclude(
        category=InsurancePremium.Category.EARTHQUAKE,
    )
    if exclude_year_end_adjusted:
        qs = qs.filter(submitted_in_year_end_adjustment=False)

    def sum_by(cat, contract):
        return qs.filter(category=cat, contract_type=contract).aggregate(
            s=Sum('annual_amount'),
        )['s'] or 0

    cats = [
        InsurancePremium.Category.LIFE_GENERAL,
        InsurancePremium.Category.LIFE_CARE_MEDICAL,
        InsurancePremium.Category.LIFE_ANNUITY,
    ]
    per_category = {}
    raw_sum = 0
    for cat in cats:
        new_total = sum_by(cat, InsurancePremium.ContractType.NEW)
        old_total = sum_by(cat, InsurancePremium.ContractType.OLD)
        deduction = calc_life_category(new_total, old_total)
        per_category[cat] = {
            'new_paid': new_total,
            'old_paid': old_total,
            'deduction': deduction,
        }
        raw_sum += deduction

    return {
        'per_category': per_category,
        'raw_sum': raw_sum,
        'total': min(raw_sum, 120_000),  # 合算上限 12 万円
    }
```

### 地震保険料控除（上限 5 万円）

```python
def calc_earthquake_total(year: int, exclude_year_end_adjusted: bool = False) -> dict:
    qs = InsurancePremium.objects.filter(
        year=year,
        category=InsurancePremium.Category.EARTHQUAKE,
    )
    if exclude_year_end_adjusted:
        qs = qs.filter(submitted_in_year_end_adjustment=False)

    paid = qs.aggregate(s=Sum('annual_amount'))['s'] or 0
    return {
        'paid': paid,
        'deduction': min(paid, 50_000),
    }
```

### 統合エントリポイント

```python
def calculate_insurance_deduction(year: int, exclude_year_end_adjusted: bool = False) -> dict:
    """生命保険料控除 + 地震保険料控除 のまとめ計算"""
    life = calc_life_total(year, exclude_year_end_adjusted)
    earthquake = calc_earthquake_total(year, exclude_year_end_adjusted)
    return {
        'life': life,           # 生命保険料控除（合算上限 12 万円適用後）
        'earthquake': earthquake,  # 地震保険料控除（上限 5 万円適用後）
        'grand_total': life['total'] + earthquake['deduction'],
        'exclude_year_end_adjusted': exclude_year_end_adjusted,
    }
```

### 計算検算例（テストで網羅）

| ケース | 入力 | 期待出力 |
|---|---|---|
| 新契約のみ、20,000 円 | LIFE_GENERAL new 20,000 | 控除 20,000 |
| 新契約のみ、40,000 円 | LIFE_GENERAL new 40,000 | 控除 30,000 (= 40,000/2 + 10,000) |
| 新契約のみ、80,000 円 | LIFE_GENERAL new 80,000 | 控除 40,000 (上限) |
| 新契約のみ、100,000 円 | LIFE_GENERAL new 100,000 | 控除 40,000 (上限) |
| 旧契約のみ、25,000 円 | LIFE_GENERAL old 25,000 | 控除 25,000 |
| 旧契約のみ、50,000 円 | LIFE_GENERAL old 50,000 | 控除 37,500 (= 50,000/2 + 12,500) |
| 旧契約のみ、100,000 円 | LIFE_GENERAL old 100,000 | 控除 50,000 (上限) |
| 新旧混在 (有利選択) | new 30,000 + old 60,000 | max(計算結果) を採用 |
| 介護医療×旧契約 | LIFE_CARE_MEDICAL old | ValidationError |
| 全枠満額 | 各枠 80,000 新 × 3 = 控除 40,000×3 = 120,000 | 上限 120,000 |
| 旧契約 50,000 全枠 + 介護医療新 80,000 | 一般 old 50,000 (37,500) + 介護新 80,000 (40,000) + 年金 old 50,000 (37,500) = raw 115,000 | 115,000 (12 万円未満) |
| 地震 30,000 | EARTHQUAKE 30,000 | 控除 30,000 |
| 地震 80,000 | EARTHQUAKE 80,000 | 控除 50,000 (上限) |
| 年調除外 | 年調済を 1 件作成、`exclude_year_end_adjusted=True` | その 1 件が計算から外れる |

## DB 影響

- **migration**: `0015_insurance_premium.py`（新規テーブル 1 個のみ、破壊的変更なし）

- **既存運用との関係**: 保険料の引落取引（`tax_tag=OTHER_DEDUCTIBLE` 等で運用中の場合）は従来通り。本機能は独立計算で動く

## セキュリティ / データ保護

- 全 view `@login_required`（Cloudflare Access 前段 + Django ログインの二重認証は維持）
- 入力値は `InsurancePremiumForm` で Django form validation を通す
- `insurer` / `policy_number` は CharField（template autoescape 前提、`safe` 不使用）
- CSV injection 対策: `csv_safe_row` 経由
- `InsurancePremium.delete` は POST のみ受付（CSRF 保護）
- AuditLog: 新規 source `insurance_premium_create` / `insurance_premium_update` / `insurance_premium_delete` を追加
- **個人情報の取扱い**:
  - `policy_number`（証券番号）は契約者本人の識別情報。CSV ダウンロード以外で外部送信されない確認
  - 控除証明書原本は紙保管（本アプリは数値のみ保持、PDF 添付機能は v1.17 では実装しない）

### v1.17.0 では実装しない
- **旧長期損害保険料控除** — 2006/12/31 以前契約継続中の長期損害保険のみ。利用者が限られる。需要発生時に v1.17.1 等で追加検討
- **控除証明書 PDF 添付** — 紙保管前提。スキャン保管したい需要が出てから検討
- **世帯メンバー別の保険料管理** — 当面は契約者 1 名分のみ。世帯メンバー別管理が必要になったら拡張

```

2. /insurance-premiums/?year=YYYY → 「+ 保険料を追加」
3. 控除証明書を見ながら以下を入力:
   - 区分 (一般 / 介護医療 / 個人年金 / 地震)
   - 契約区分 (新 / 旧)  ※ 介護医療は新固定、地震は不問
   - 保険会社名、証券番号、年間払込予定額（申告額）
   - 「年末調整で提出済」: 給与所得者で会社に提出した分は ON

   申告書に転記すべき額を確認 → CSV ダウンロード
6. 控除額カードの数値を申告書第二表 ⑮生命保険料控除 / ⑯地震保険料控除 に記入
```

## レビュー確定事項 (2026-05-14)

| ID | 論点 | 決定 | 理由 |
|---|---|---|---|
| UA-1 | 旧長期損害保険料控除の包含 | **非対象** | 対象家庭が限定的。需要発生時に v1.17.1 で追加検討 |
| UA-2 | 共済の扱い | **現案採用** | 控除証明書記載通り（`insurer` に組合名、区分も証明書通り）。OPERATIONS_GUIDE で明記 |
| UA-3 | 年間支払保険料の入力単位 | **円単位（整数）** | 控除証明書は円単位記載。`//` で端数切捨て |
| UA-4 | 年セレクトの範囲 | **現在年 ± 2 年** | 初年度の入力導線確保 |
| UA-5 | v1.18.0 統合方針 | **v2 を別 URL で作る** | v1.13.0 / v1.16.0 完全後方互換維持 |
| UA-6 | 年単位 unique 制約 | **入れない** | 同一会社で複数契約あり得る。重複は block しない |

### 補足: スコープ明示（実装時の混乱防止）

以下は v1.17.0 では**対象外**:
- 国民年金基金、小規模企業共済 等 → **保険料控除ではなく** 社会保険料控除 / 小規模企業共済等掛金控除に該当
- 国民健康保険料、国民年金保険料 → 同上
- 厚生年金 → 同上
- 介護保険料（40 歳以上の社会保険） → 同上

v1.17.0 が扱うのは **「生命保険料控除証明書」「地震保険料控除証明書」が発行される民間保険のみ**。
