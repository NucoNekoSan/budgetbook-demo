# v1.15.0 Auto Payoff Projection (B/S 各負債口座行)

## 目的

`/balance-sheet/` の各負債口座行に、**現在の残高 + LoanProfile.monthly_payment + 年利** を前提とした **完済予測** を表示する。`/loan-strategy/` ページが「全体最適のシミュレーション (avalanche/snowball + 繰上額)」を扱うのに対し、本機能は **各口座が単体でどう推移するか** を見せる。

ユーザは現在の月次返済額のままで「あと N ヶ月で完済」「想定総利息 ¥X」が即座に分かるようになる。

`docs/RIBO_OPERATION_FEATURE_MEMO.md` #2 Auto Payoff Projection の実装。

## スコープ

### 対象

- 新規サービス `ledger/services/loan_projection.py` に `project_fixed_principal_payoff(account, as_of=None, owed=None, profile=None) -> dict | None` を追加
- `services/balance.py::balance_sheet()` の各 liability item に `projection: dict | None` を付与（既存フィールドは破壊しない）
- `templates/ledger/balance_sheet.html` の負債口座行に「あと N ヶ月 / 完済予定 YYYY-MM / 想定総利息」を追加表示
- `LoanProfile.monthly_payment` が 0 or 未設定の口座、`owed <= 0` の口座は projection 無し

### 非対象

- 繰上返済 (`monthly_extra`) のシナリオ — `/loan-strategy/` 側で既に提供
- 戦略比較 (avalanche / snowball) — `/loan-strategy/` の領分
- LoanProfile.payoff_date (手入力) の自動更新 — 手入力欄は併記する
- 管理コマンド化 — 表示専用、CLI 不要
- DB migration — read-only 計算

## 計算ロジック

口座単体のシミュレーション。`MAX_MONTHS = 600` で打切り (loan_strategy.py と整合)。

```text
owed = -all_account_balances(as_of)[account.pk]   # 正値
rate = annual_rate_bp / 10000 / 12                 # 月利
payment = monthly_payment                          # 銀行から引かれる総額 (v1.12.0 確定)

if annual_rate_bp == 0:
    months_remaining = ceil(owed / payment)
    total_interest = 0
else:
    # 月毎: 利息加算 → payment 支払 → 残債更新
    # 最終月は余り分のみ支払 (端数調整)
    if payment <= int(owed * rate):
        # 利息に届かない → 永遠に減らない
        return {warning: 'payment_below_interest', months_remaining: None, ...}
    # ループで months_remaining を計算
```

`next_interest = int(owed * rate)`, `next_principal = payment - next_interest` を「次回引落」情報として返す。

`next_payment_date`: `LoanProfile.payment_day` (1-31, 0 は月末) を使い、`as_of` 以後で最も近い該当日。`as_of` 当日が `payment_day` ならその日を返す。

`payoff_date`: `as_of` から `months_remaining` ヶ月後の `next_payment_date` を進めた日付。`payment_day == 0` (月末) の場合は最終月の月末。`payment_day` 未設定 (`None`) の場合は `None`。

## テスト計画

`ledger/tests/test_loan_projection.py` を新規作成。

| ケース | 期待 |
|---|---|
| 利息ありローン (16.8%, 残¥388,897, 月¥10,000) | months_remaining が概ね 52〜60、total_interest > 0、warning なし |
| 利息ゼロローン (0%, 残¥34,300, 月¥4,900) | months_remaining = 7、total_interest = 0 |
| monthly_payment ≤ 月利息 (例: 残¥100,000, 18%, 月¥1,000 < 月利息¥1,500) | warning='payment_below_interest', months_remaining=None |
| LoanProfile 無し | return None |
| monthly_payment = 0 | return None |
| owed <= 0 (既に完済) | return None |
| as_of=過去日 → その時点残高ベースで計算 | 直近取引が反映されている |
| 最終月の端数調整 | total_paid ≈ owed + total_interest |

`test_balance_sheet.py` にも 1 件追加:
- LoanProfile 付き負債口座を含む B/S で `liabilities[i]['projection']` が dict を持つ
- 手入力 `payoff_date` がある口座でも `projection` は別途計算される

## 表示

`templates/ledger/balance_sheet.html` の負債口座行 (line 70-94 周辺) に追加:

```html
{% if item.projection %}
  {% if item.projection.warning == 'payment_below_interest' %}
    <div class="meta amount-expense">⚠ 月次返済額が利息に届かず元金が減りません</div>
  {% elif item.projection.months_remaining %}
    <div class="meta">
      自動完済予測: あと {{ item.projection.months_remaining }} ヶ月
      ({{ item.projection.payoff_date|date:'Y-m' }})
      / 想定総利息 {{ item.projection.total_interest|yen }}
    </div>
  {% endif %}
{% endif %}
```

既存の手入力 `payoff_date` 表示はそのまま残す（ユーザの目標と自動予測の比較が可能）。

## セキュリティ影響

- 計算は read-only、DB 書込なし
- 入力は既存モデル (`Account`, `LoanProfile`) のみ、外部入力ナシ
- XSS: Django autoescape のみで安全 (`safe` フィルター不使用)
- CSV / 公開出力: なし

## DB / データ影響

- migration なし
- 既存テーブル参照のみ
- 既存テストは引き続き緑であること (新フィールド追加は dict のキー追加のみ、破壊なし)

`git revert` で完結。DB 変更なしのため副作用なし。
