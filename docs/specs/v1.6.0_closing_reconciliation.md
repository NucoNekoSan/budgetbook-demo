# v1.6.0 Monthly Closing And Reconciliation

## 目的

家計簿を「入力しただけの記録」から、月次で締め、実口座残高と照合できる状態へ引き上げる。

## 仕様

- 月次締めでは、対象月、月初繰越、収入、支出、月末残高、口座別残高スナップショットを保存する。
- 1 か月につき締めレコードは 1 件のみ。
- 締め済み月の通常取引・振替は、作成・更新・削除をサーバー側で拒否する。
- 月次締めが存在する場合、口座の初期残高変更は拒否する。
- 口座残高照合では、照合日、口座、帳簿残高、実残高、差額、メモを保存する。
- 帳簿残高はサーバー側で計算し、ユーザー入力を信用しない。
- 既存 `Transaction`, `Transfer`, `Account`, `Category` のスキーマは変更しない。

## モデル

### MonthlyClosing

- `month`: DateField(unique)。必ず月初日。
- `closed_at`: DateTimeField(auto_now_add)
- `closed_by`: User FK nullable
- `opening_carry`: IntegerField
- `income`: IntegerField
- `expense`: IntegerField
- `net`: IntegerField
- `closing_balance`: IntegerField
- `account_balances`: JSONField
- `notes`: TextField(blank)

### AccountReconciliation

- `account`: Account FK PROTECT
- `reconciled_on`: DateField
- `book_balance`: IntegerField
- `actual_balance`: IntegerField
- `difference`: IntegerField
- `notes`: TextField(blank)
- `created_by`: User FK nullable
- `unique(account, reconciled_on)`

## 画面

- `/accounting/` を追加する。
- 月次締めフォームと、直近の締め一覧を表示する。
- 口座残高照合フォームと、直近の照合一覧を表示する。

## 受け入れ条件

- 月次締めを作成すると、対象月の集計スナップショットが保存される。
- 同じ月を二重に締められない。
- 締め済み月の取引作成・更新・削除は拒否される。
- 月次締め後に口座初期残高を変更できない。
- 口座照合を作成すると、帳簿残高はサーバー側計算値になる。
- 実残高との差額が保存される。
- 既存テーブルへの破壊的 migration がない。
- `python manage.py test ledger` が通る。

## 非対象

- 銀行 API 連携。
- レシート画像添付。
- 既存取引の自動カテゴリ変更。
- 締め解除画面。必要な場合は Django admin で慎重に処理する。

## ロールバック

- `migrate ledger 0004` へ戻すと追加テーブルは削除される。
- 既存の取引・振替・口座・カテゴリデータには影響しない。
- 締め・照合レコードは失われるため、適用前バックアップを必須とする。
