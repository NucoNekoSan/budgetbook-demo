# v1.2.0 Transfer And Carryover

Status: 実装完了

## 目的

複数口座間のお金の移動が、収入・支出として集計される問題を解消する。あわせて、月初繰越・当月収支・月末残高をダッシュボードで確認できるようにする。

## 採用設計

`Transaction` は既存の収入・支出取引用として維持し、新規に `Transfer` モデルを追加する。

理由:

- 既存 `Transaction` データを移行しないため、実運用 DB への影響が最小。
- 既存の `category__kind` ベースの収入・支出集計を壊しにくい。
- 口座残高計算で、通常取引と振替を明確に分けられる。

## モデル仕様

新規 `Transfer`:

- `date`: DateField
- `from_account`: ForeignKey(Account, PROTECT, related_name=`transfers_out`)
- `to_account`: ForeignKey(Account, PROTECT, related_name=`transfers_in`)
- `amount`: IntegerField, 1 以上
- `description`: CharField(max_length=120)
- `memo`: TextField(blank=True)
- `created_at`, `updated_at`: `TimeStampedModel` を継承

Validation:

- `from_account == to_account` はエラー。
- `amount < 1` はエラー。

## 集計仕様

収入・支出:

- `Transaction` のみで計算する。
- `Transfer` は収入合計、支出合計、支出カテゴリ集計、収支推移に含めない。

口座別残高:

```text
balance =
  opening_balance
  + income_transactions_until_date
  - expense_transactions_until_date
  - transfers_out_until_date
  + transfers_in_until_date
```

全口座合計:

- 振替の出金と入金は相殺される。
- 振替によって全口座合計残高は変化しない。

月次繰越:

- 月初繰越 = 対象月の前日までの全口座合計残高。
- 当月収支 = 当月収入 - 当月支出。
- 月末残高 = 対象月末時点の全口座合計残高。

## 画面仕様

### 右側フォーム

- フォーム種別として「通常取引」「振替」を切り替えられるようにする。
- 通常取引は既存 `TransactionForm` を使う。
- 振替は新規 `TransferForm` を使う。
- 通常取引内の「収入 / 支出」選択とカテゴリ絞り込みは既存仕様を維持する。

### ダッシュボード

- 月初繰越、当月収支、月末残高を表示する。
- 口座残高には振替を反映する。
- 取引一覧には `Transaction` と `Transfer` を時系列で統合表示する。
- 振替行は中立表示にし、カテゴリ欄は「振替」、口座欄は「出金元 -> 入金先」とする。

### フィルター

- 摘要検索 `q`: `Transaction.description` と `Transfer.description` の両方に適用する。
- 口座フィルター `account`: `Transaction.account`, `Transfer.from_account`, `Transfer.to_account` のいずれかに一致するものを表示する。
- カテゴリフィルター `category`: `Transaction` のみに適用し、`Transfer` は除外する。

### CSV

- 対象月の `Transaction` と `Transfer` を日付順で出力する。
- 振替行は種別を「振替」として識別できるようにする。
- 口座列は「出金元 -> 入金先」、カテゴリ列は「振替」とする。

## URL

- `transfers/new/` -> `transfer_create`
- `transfers/<int:pk>/edit/` -> `transfer_update`
- `transfers/<int:pk>/delete/` -> `transfer_delete`

## Admin

`Transfer` を Django admin に登録する。

`list_display`:

- `date`
- `description`
- `from_account`
- `to_account`
- `amount`

`list_filter`:

- `from_account`
- `to_account`
- `date`

`search_fields`:

- `description`
- `memo`

## テスト観点

必須:

- 振替が収入合計に含まれない。
- 振替が支出合計に含まれない。
- 振替が支出カテゴリ別集計に含まれない。
- 振替で出金元口座の残高が減る。
- 振替で入金先口座の残高が増える。
- 振替で全口座合計残高が変わらない。
- 同一口座への振替が validation error になる。
- 月初繰越と月末残高が正しい。
- 既存の収入・支出取引作成と更新が壊れていない。
- 口座フィルターで `from_account` 側の振替が表示される。
- 口座フィルターで `to_account` 側の振替が表示される。
- カテゴリフィルター時に振替が除外される。
- 振替編集で残高計算が更新される。
- 振替削除で残高計算が戻る。
- CSV export に振替行が含まれる。

## 非対象

- `Transaction` のスキーマ変更。
- 既存 `Transaction` データの変換。
- v1.3.0 の支出比率グラフ。
- v1.4.0 のインライン編集 UX。
- v1.5.0 の Docker / Cloudflare 対応。

## ロールバック注意

`Transfer` 登録後に `ledger` migration を `0002` へ戻すと、振替データは失われる。既存 `Transaction`, `Account`, `Category` は影響を受けない。
