# v1.18.0 確定申告レポート v2（統合）

> **Status**: ✅ **確定済（2026-05-14）**
> 起草・レビュー: 2026-05-14

## 目的

v1.16.0（医療費控除）、v1.17.0（生命保険料控除・地震保険料控除）、v1.13.0（税控除タグ集計）で個別管理してきた控除データを、**確定申告書 第二表「所得から差し引かれる金額に関する事項」の物理レイアウトに沿った 1 ページ統合レポート**として表示する。

利用者は v2 レポートを開けば、医療費 → 生命保険料 → 地震保険料 → 寄附金の順で申告書記入欄ごとに「ここに ¥X を書く」という形で控除額を確認でき、CSV 1 ファイルで全控除のサマリをエクスポートできる。

v1.13.0 / v1.16.0 / v1.17.0 の各画面は **後方互換のため残置**。v2 はあくまで統合ビュー。

## 設計（合意済み）

### ① URL: 新 URL を別途用意（既存改造なし）

- `/reports/tax-deductions/v2/?year=YYYY`（HTML）
- `/reports/tax-deductions/v2.csv?year=YYYY`（CSV）
- v1.13.0 の `/reports/tax-deductions/` は **無変更**（v1 として併存）

### ② スコープ（A 案採用）

統合対象:
- **医療費控除** — v1.16.0 の `calculate_medical_deduction()` を呼ぶ
- **生命保険料控除** — v1.17.0 の生保 3 枠合算（上限 12 万円）
- **地震保険料控除** — v1.17.0 の地震（上限 5 万円）
- **寄附金関連（参考表示）** — v1.13.0 の `tax_tag=donation` 集計年間支払合計のみ

寄附金は **控除額計算ロジックを実装しない**（ふるさと納税ワンストップ特例運用前提、所得税控除/住民税控除の選択計算は v1.19.0 以降で必要時に追加）。表示は「年間支払合計」+ 注記「寄附金控除の正確な金額は確定申告書 B 様式の計算欄を参照」。

### ③ 申告書転記支援テキスト

各セクションのカード末尾に、対応する申告書の欄番号を明示:
- 医療費控除: 「申告書 第二表 ⑩医療費控除 = ¥XXX,XXX」
- 生命保険料控除: 「申告書 第二表 ⑮生命保険料控除 = ¥XXX,XXX」
- 地震保険料控除: 「申告書 第二表 ⑯地震保険料控除 = ¥XX,XXX」
- 寄附金: 「申告書 第二表 ⑲寄附金控除 ＝ （別途計算が必要）」

※ 欄番号は 2026 年現在の様式に基づく。様式改訂時は手動更新（NOTE コメント付与）。

### ④ 年末調整提出済を含む/除外の切替

生命保険・地震保険セクションは v1.17.0 の `submitted_in_year_end_adjustment` を尊重。デフォルトは **「年調済を除外」（=確定申告で書く額）** で表示する。トグルで切替可能。

医療費・寄附金は年調対象外のため切替に影響しない。

【医療費控除 合計】, ¥XX,XXX

【生命保険料控除】
... 内訳行 ...

【地震保険料控除】
... 明細行 ...
【地震保険料控除 合計（申告書 ⑯）】, ¥XX,XXX

【寄附金（参考）】
... 取引一覧 ...
【寄附金 年間支払合計（参考、申告書 ⑲は別途計算）】, ¥XX,XXX

UTF-8 BOM + `csv_safe_row` injection 対策（既存ユーティリティ流用）。

## スコープ

### サービス層

新規 `budgetbook/ledger/services/tax_report_v2.py`:

```python
@dataclass
class TaxReportV2Summary:
    year: int
    exclude_year_end_adjusted: bool
    medical: MedicalDeductionSummary       # v1.16.0
    insurance: InsuranceDeductionSummary   # v1.17.0
    donation: dict                          # {'total': int, 'count': int, 'transactions': [...]}
    grand_deduction_total: int              # medical.deduction + insurance.grand_total
                                            #（寄附金は控除額未計算のため除外）


def build_tax_report_v2(year, exclude_year_end_adjusted=True) -> TaxReportV2Summary:
    medical = calculate_medical_deduction(year)
    insurance = calculate_insurance_deduction(year, exclude_year_end_adjusted)
    donation = _aggregate_donation(year)
    grand = medical.deduction + insurance.grand_total
    return TaxReportV2Summary(...)
```

`_aggregate_donation(year)` は `Transaction.objects.filter(date__year=year, category__tax_tag='donation', category__kind='expense')` の集計。

## DB 影響

**なし**。新規テーブル・新規カラム・データ変換いずれも無し。読み取り専用の集計レポート機能のみ。

- migration: なし
- バックアップ手順: 不要（読み取り専用）
- ロールバック: コード revert のみで完全に戻る

- 申告書様式の自動 PDF 生成
- 国税庁 e-Tax への直接送信
